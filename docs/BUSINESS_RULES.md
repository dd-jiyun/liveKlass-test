# 수강 신청 시스템 - 비즈니스 규칙

## 1. 강의 관리

### 상태 전이

```
DRAFT → OPEN → CLOSED → DRAFT (→ OPEN 재공개)
```

| 상태 | 설명 | 신청 가능 여부 |
|------|------|:---:|
| DRAFT | 초안, 미공개 | 불가 |
| OPEN | 모집 중 | 가능 |
| CLOSED | 모집 마감 | 불가 |

> CLOSED 후 재공개 흐름: `CLOSED → DRAFT (reopen)` → 마감일 수정 → `DRAFT → OPEN (open)`

### 상태 전환 규칙

**DRAFT → OPEN**
- 크리에이터 수동 전환
- 신청 마감일이 현재 날짜 이후여야 함. 미충족 시 400

**OPEN → CLOSED**
- 신청 마감일 도달 시 스케줄러가 자동 처리
- 크리에이터 수동 전환
- 수동 마감 시 기존 대기 중 / 알림 받은 대기자 전원 자동 취소

**CLOSED → DRAFT (재공개 준비)**
- 크리에이터 수동 전환 — CLOSED 강의를 초안으로 되돌려 마감일을 수정할 수 있는 상태로 복원
- DRAFT 상태로 돌아간 뒤 신청 마감일을 수정하고 다시 OPEN 전환

**DRAFT → OPEN (재오픈)**
- 동일하게 신청 마감일이 현재 날짜 이후여야 오픈 가능
- 재오픈 시 기존 대기 순번 없음 — 선착순으로 새로 신청

> 정원 초과는 강의 마감 트리거가 아닙니다. 정원이 차도 강의는 OPEN 유지되며, 신청 시도 시 409로 거부합니다.

### 강의 등록 필드 및 유효성 제약

| 필드 | 제약 | 변경 가능 시점 |
|------|------|------|
| 제목 | 필수, 공백 불가 | 초안(DRAFT)만 |
| 설명 | 필수 | 초안(DRAFT)만 |
| 수강료 | 0 이상 (0이면 무료) | 초안(DRAFT)만 |
| 최대 정원 | 1 이상 | 초안(DRAFT)만 |
| 강의 시작일 | 필수 | 초안(DRAFT)만 |
| 강의 종료일 | 시작일 이후 | 초안(DRAFT)만 |
| 신청 마감일 | 시작일 이전 | 초안(DRAFT) + 재오픈 시 |
| 취소 가능 기간(일) | 0 이상 (0이면 취소 불가) | 초안(DRAFT)만 |

> OPEN 이후 핵심 필드 변경 불가 — 이미 신청한 수강생의 조건이 바뀌면 분쟁 소지.
> 신청 마감일만 예외적으로 재오픈 시 연장 가능.

## 2. 수강 신청 관리

### 상태 전이

```
PENDING → CONFIRMED
       ↘ CANCELLED (만료 자동 or 수동 취소)
```

| 상태 | 설명 |
|------|------|
| PENDING | 신청 완료, 결제 대기 (임시 좌석 점유 중) |
| CONFIRMED | 결제 완료, 수강 확정 |
| CANCELLED | 취소됨 |

### PENDING — 임시 좌석 점유

- 수강 신청 성공 시 즉시 PENDING 생성 + 수강 인원 +1
- 만료 시간: **20분** (기차표/시험 신청 업계 표준)
- 20분 내 결제 미완료 시 자동 취소 + 수강 인원 -1 → 대기 1순위 알림

### 결제 확정 (CONFIRMED)

- PENDING 상태에서만 전환 가능. 만료된 경우 불가
- 외부 결제 시스템 연동 없이 단순 상태 변경으로 대체
- 무료 강의(수강료 0원): PENDING 생성 직후 시스템이 자동 CONFIRMED 전환
  - 이력: 사유 `PAYMENT_CONFIRMED`, 처리자 `SYSTEM`

### 중복 신청 방지

- 동일 강의에 PENDING 또는 CONFIRMED 상태가 이미 존재하면 거부 (409)
- CANCELLED 상태는 재신청 허용 (만료 후 재신청, 재수강, 재오픈 후 신청 포함)

### 수강 취소 (CANCELLED)

**임시 예약(PENDING) 수동 취소**
- PENDING 상태에서 사용자가 직접 취소 가능
- 즉시 취소 + 수강 인원 -1 → 대기 1순위 알림
- 20분 만료 대기 없이 즉시 자리 반환 (사용성 우선)

**수강 확정(CONFIRMED) 수동 취소**
- 취소 가능 기간: **강의 시작일 기준 N일 전까지** (기본값 7일, 크리에이터 설정 가능)
- 기간 초과 시 거부
- 취소 성공 시 수강 인원 -1 → 대기 1순위 알림

## 3. 정원 관리 및 동시성 제어

### 수강 인원 변화 규칙

| 이벤트 | 수강 인원 |
|--------|:---:|
| PENDING 생성 | +1 |
| PENDING → CONFIRMED | 변화 없음 |
| PENDING 만료 → CANCELLED | -1 |
| CONFIRMED → CANCELLED | -1 |

### 수강 신청 처리 흐름 (비관적 락)

```
POST /enrollments  (일반 수강 신청)
1. 강의 row에 데이터베이스 잠금 (SELECT FOR UPDATE)
2. 이미 PENDING/CONFIRMED 신청이 존재?
   → YES: 409 (중복 신청 거부)
3. 수강 인원 < 최대 정원?
   → YES: 다음 단계
   → NO : 409 (정원 초과, 대기 등록 안내)
4. 이 강의에 알림 받은 대기자가 존재?
   → YES: 409 (대기자 우선 예약 중)
5. PENDING 생성 + 수강 인원 +1
6. 트랜잭션 종료 → 잠금 해제
```

> **알림 받은(NOTIFIED) 대기자의 수강 신청은 별도 엔드포인트를 사용합니다.**
> `POST /waitlists/{id}/convert` — 대기 상태를 CONVERTED로 전환 후 PENDING 생성.
> NOTIFIED 대기자가 `POST /enrollments`를 호출하면 Step 4에서 409로 차단됩니다.

- 수강 인원 컬럼으로 전체 집계 없이 빠른 정원 검증
- 알림 받은 대기자 우선권: 20분 동안 일반 신청 차단
- 잠금 범위는 해당 강의 row로 최소화

## 4. 대기열 관리

### 대기 등록 조건

- 사용자가 명시적으로 대기 등록을 요청 — 신청 실패 시 자동 등록 없음
- 정원 초과 거부(409) 응답에 "대기 등록 가능" 안내 포함
- 등록 가능 조건: 강의 OPEN 상태 + 수강 인원이 정원 이상
- CLOSED 강의는 대기 등록 불가
- 동일 강의에 대기 중 또는 알림 받은 상태가 이미 존재하면 거부 (409)

### 대기열 상태

| 상태 | 설명 |
|------|------|
| WAITING | 대기 중 |
| NOTIFIED | 신청 기회 부여됨 (20분 수락 대기) |
| EXPIRED | 수락 시간 내 미응답으로 만료 |
| CONVERTED | 수강 신청으로 전환됨 |
| CANCELLED | 대기 취소 |

### 자리 배정 흐름

```
자리 발생 (임시 예약 만료 or 수강 확정 취소)
    → 수강 인원 -1
    → 대기 1순위: WAITING → NOTIFIED (20분 수락 시간 시작)
    → 수락 (수강 신청 요청): PENDING 생성, 대기 상태 CONVERTED
    → 미응답 (20분 만료): EXPIRED → 다음 순번 NOTIFIED
```

- 수락 시간: **20분** (임시 예약 만료 시간과 동일)
- 대기 순번: 등록 시각 기준 FIFO
- 알림 수단: 알림 시각을 DB에 기록. 알림 발송 레이어는 인터페이스로 추상화해 확장 가능한 구조 유지

## 5. 감사 로그 (EnrollmentHistory)

수강 신청 상태가 바뀔 때마다 이력을 INSERT합니다. 수정/삭제는 불가하며, 수강 신청 상태 변경 트랜잭션 내에서 반드시 함께 기록합니다.

기록 대상: 수강 신청, 결제 확정, 사용자 취소, 임시 예약 만료. 처리 주체(사용자 / 시스템)와 사유를 함께 저장합니다.

> 테이블 구조는 [ERD.md](ERD.md) 참고.

## 6. 전체 상태 흐름 요약

```
[수강 신청 플로우]
  신청 요청
    → 알림 받은 대기자? → PENDING (대기 CONVERTED)
    → 다른 알림 받은 대기자 존재? → 409 (대기자 우선)
    → 정원 있음? → PENDING (20분 타이머)
    → 정원 초과? → 409 → 대기 등록 → WAITING

  PENDING
    → 결제 완료 → CONFIRMED
    → 무료 강의 → 즉시 자동 CONFIRMED
    → 20분 만료 → CANCELLED → 대기 1순위 알림

  CONFIRMED
    → 취소 가능 기간 내 취소 → CANCELLED → 대기 1순위 알림
    → 기간 초과 취소 시도 → 거부

[강의 상태 흐름]
  DRAFT  → (크리에이터, 마감일 > 오늘 검사) → OPEN
  OPEN   → (마감일 도달 or 크리에이터 수동) → CLOSED
  CLOSED → (크리에이터 reopen 요청) → DRAFT → (마감일 연장 후 open) → OPEN
```
